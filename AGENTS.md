# KnowledgeRepo 结构说明

这个仓库采用受 Andrej Karpathy 的“LLM Knowledge Bases”思路启发的知识库工作流：

1. 把原始资料放入 `原始资料/`。
2. 使用 LLM 持续整理并维护 `知识库/` 下的 Markdown 知识库。
3. 把问答、报告、幻灯片、图表等产出放入 `派生内容/`。
4. 把辅助脚本放入 `工具/`，把一致性检查和健康检查放入 `检查/`。

## 目录结构

### 原始资料/
存放导入仓库的原始资料。

- `原始资料/文章/`：保存网页文章、博客、新闻、长文，可为 Markdown、HTML 或文本。
- `原始资料/论文/`：保存论文、论文笔记、文献摘录。
- `原始资料/仓库/`：保存代码仓库快照、仓库说明、源码笔记。
- `原始资料/数据集/`：保存 CSV、JSON、表格和数据说明。
- `原始资料/图片/`：保存与资料相关的图片、图表、截图。
- `原始资料/碎片记录/`：保存尚未成文的零散想法、摘抄、聊天片段、观察记录、语音转写和临时笔记。
- `原始资料/网页剪藏/`：保存从浏览器或其他工具手动剪藏的页面资料。

### 知识库/
由 LLM 维护的 Markdown 知识库主体。

- `知识库/索引/`：入口页、总览页、导航页、地图页。
- `知识库/概念/`：概念定义、术语解释、标准条目。
- `知识库/主题/`：围绕某个主题串联多个概念和来源的综述页。
- `知识库/来源笔记/`：针对单个来源形成的笔记，记录候选概念、候选主题，以及已编入的知识库条目并尽量显式回链。
- `知识库/摘要/`：简明摘要、读书笔记、阶段性总结。

### 派生内容/
由知识库或查询任务生成的结果文件。

- `派生内容/回答/`：针对问题直接生成的 Markdown 回答。
- `派生内容/报告/`：较长的研究报告、调研简报、对比分析。
- `派生内容/幻灯片/`：演示文稿，例如 Marp Markdown。
- `派生内容/可视化/`：图表、流程图、渲染图片、分析图。

### 检查/
存放知识库健康检查、数据审计、一致性检查结果。

### 工具/
存放人或代理会使用的辅助工具。

- `工具/命令行/`：小型命令行工具。
- `工具/脚本/`：一次性脚本或批处理脚本。

## 使用规则

- 优先把新资料放进 `原始资料/`，而不是直接手改 `知识库/`。
- 把 `知识库/` 视为由 LLM 持续整理和编译出的结果层。
- 当某个派生结果已经足够稳定、值得长期复用时，再把它整理回 `知识库/`。
- 报告、实验、中间问答结果优先放在 `派生内容/`，不要直接混入主知识库。
- 来源笔记应区分“候选概念 / 候选主题”和“已编入条目”；对已存在条目优先使用显式 Markdown 链接，而不是只留纯文本词项。
- 仓库结构发生变化时，同步更新本文件。
